# Contributing Guidelines

Welcome to Code Cortex Portfolio Web App!  
Thank you for contributing to this project. Please follow these guidelines to ensure smooth collaboration.

---

## Branch Naming

- **Features:** `feature/<description>`  
- **Bug fixes:** `bugfix/<description>`  
- **Hotfixes:** `hotfix/<description>`  

---

## Pull Request Workflow

1. Create a **new branch** from `main`  
2. Work on your feature or fix  
3. Commit changes with **clear messages**  
4. Push branch to GitHub  
5. Open a **Pull Request (PR) to `main`**  
6. Wait for review and approval before merging  

---

## Commit Message Guidelines

- Use **present tense**: `Add portfolio API`, `Fix header styling`  
- Keep messages **short and clear**  
- Reference **issues if applicable**: `#1`

## Backups

- Code is versioned in GitHub  
- Keep a **weekly backup** of the repository if needed  

---

## Code Style & Quality

- Follow consistent **indentation & formatting**  
- Write **clean, readable code**  
- Add **comments** for complex logic  
- Include **unit tests** if possible  

---

## Reporting Issues

- Use the **Issues** tab to report bugs or request features  
- Provide **clear reproduction steps** and screenshots if needed
